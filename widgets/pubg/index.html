PUBG Widgets — lokale Demo

Übersicht aller PUBG-Widgets. Jedes Widget läuft als eigene Browser-Source — hier in iframes kombiniert für schnelles Sichten. Die Session aggregiert alle Matches seit sessionStartedAt (Default: Epoch, also alle DB-Matches). Reset via Button im flyout-full oder POST /api/pubg/session/reset.

▶ Gameplay scene · live, hide-on-no-session live-bar.html — top, permanent strip news-ticker.html — bottom, snippet rotator flyout-full.html — side panel, multi-section chat-stats-popup.html — top-right, !stats trigger ▶ Bot-Left rotation slots · 560×360 · session-only mates.html?layout=carousel mates.html?layout=stack mates.html?layout=fold mates.html?layout=mosaic post-match-card.html — last match summary map-performance.html?range=session top-mates-slider.html?range=session — rotates sort categories first-fight.html — first-fight survival rate (all matches) first-fight.html?excludeHotDrop=1 — silent-drop fights only (no hot drops) chicken-map.html?range=session chicken-together.html — wins with which mates session-lobbies.html — Lobby Avg K/D + sparkline hot-drop.html — hot-drop rate, survival, streak streak-counter.html?type=chicken streak-counter.html?type=top10 session-goal.html?goal=kills:20 — progress bar session-goal.html?goal=chickens:3 trend-indicator.html — current vs. last session session-achievements.html — milestone log ▶ Starting Soon / BRB · lifetime, always-on news-ticker.html?focus=lifetime career-card.html — PUBG-API lifetime career top-mates.html?sortBy=synergy (Composite KDA — recommended) top-mates.html?sortBy=mostPlayed top-mates.html?sortBy=chickensTogether top-mates.html?sortBy=mateKd top-mates.html?sortBy=kd (Best K/D) top-mates.html?sortBy=winRate map-performance.html?range=all chicken-map.html?range=all session-summary.html map-distribution.html (candidate for removal) ▶ Stream-Ending / fullscreen session-report.html — full after-session report (phases, maps, trend, streaks, achievements) ▶ Just-Chatting / Tools lookup.html — interactive player search (PUBG-API) squad-compare.html?players=__PLAYER__&matches=5 /scenes/stats.html?player=__PLAYER__ ▶ Backend /api/pubg/status — poller health /api/pubg/db-info — first match + streamer name
